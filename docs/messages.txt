SCENARIOS
=========
Clent Connects:
	Get client ping
	Send all entities to client


CLIENT
Send avatar position and direction every 33-50ms
Render from history - 50ms ago, lerp between historical positions
	
	
SERVER
Loop every 15ms
Receive player position and direction
Run game loop
	Send entity positions for changed entities


PLAYER CONTROL
Turning is handled entirely by the client
Player moves and their avatar is moved immediately
	Client sends position and timestamp to client
	Server checks position of the client is valid
	Server sends back position of all entities
	Client corrects itself based on servers position
Player shooting
	Client sends shot msg with timestamp and direction
	Server moves players to correct position and checks the client has hit
	 
	
MESSAGES
Ping message
Create entity
Update entity
Refresh entity - for if the client doesn't know about an updated object
